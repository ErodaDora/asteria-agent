## 我们经历过的主要问题

### 1. MCP 与本地 Tool 体系如何对齐

- 问题：
  - 现有 JAgent 已有本地 tool/runtime，但没有 MCP 协议层
- 解决思路：
  - 不推翻原结构
  - 保留本地 runtime 和 tool registry
  - 补 MCP 风格的 descriptor / invocation 外壳

### 2. 小红书采集登录态不稳定

- 问题：
  - 登录窗口自动关闭
  - 误判已登录
  - 手动关闭后无法保存状态
- 解决思路：
  - 登录页改成持续保存 `storageState`
  - 关闭窗口时使用最近一次快照
  - 登录链路不再依赖脆弱的自动成功判断

### 3. Notion 同步不稳定

- 问题：
  - 出现过 `ClosedChannelException`、`ConnectException`
  - 初期标题和正文映射不理想
- 解决思路：
  - 增强异常信息
  - 调整 HTTP 请求实现与重试
  - 按数据库 schema 做字段存在判断
  - 采集结果与生成结果分别构造 payload

### 4. 前端页面重构过程里样式崩坏

- 问题：
  - 旧缓存导致页面一直显示旧样式
  - 重构时出现布局断裂
- 解决思路：
  - 重新收口成单栏居中工作流
  - 清理冗余侧栏与展示卡
  - 给静态资源加版本号，强制刷新 CSS / JS

### 5. 自动发布到小红书不稳定

- 问题：
  - 创作者登录页重复打开 / 自动关闭
  - 页面默认停在“上传视频”
  - 自动切不到“上传图文”
- 解决思路：
  - 发布链路继续复用已有 `storageState`
  - 创作者登录改为单窗口持续监控
  - 自动切图文失败时保留当前页面，允许人工接管
